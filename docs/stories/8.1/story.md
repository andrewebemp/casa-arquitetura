# Story 8.1 - LGPD Compliance: Consent Flow, Politica de Dados e Disclaimer de IA

## Status: Draft

## Story
As a user of DecorAI Brasil, I want to give explicit consent for image processing and understand how my data is used, so that the platform complies with LGPD (Lei Geral de Protecao de Dados) and I can trust that my images and personal data are handled transparently.

## Acceptance Criteria

- Given a new user completing registration (email/password or Google OAuth), when the signup form is submitted, then a mandatory LGPD consent checkbox must be checked ("Concordo com o processamento das minhas imagens conforme a Politica de Privacidade") before account creation proceeds; without consent, registration is blocked with a clear message
- Given an authenticated user, when they navigate to Settings > Privacy, then they can view their current consent status, the date consent was granted, and have the option to revoke consent (which triggers account data handling flow per LGPD Art. 18)
- Given any image generated by the AI pipeline, when the render is displayed in the UI (project view, share page, before/after slider, dashboard thumbnail), then a visible disclaimer text "Imagem ilustrativa gerada por IA" must be overlaid or displayed adjacent to the image
- Given a shared public link (Story 4.1/4.2), when a non-authenticated visitor views the shared render, then the AI disclaimer is visible on the shared page
- Given an authenticated user on Settings > Privacy, when they toggle "Permitir uso das minhas imagens para melhoria do modelo de IA", then the `training_opt_in` field is updated in their profile; the default must be `false` (opt-out by default per NFR-09)
- Given an authenticated user, when they request data export via `GET /users/me/data-export`, then the API returns a JSON file containing all their personal data, projects, renders, and preferences within 48 hours (LGPD Art. 18, V)
- Given an authenticated user, when they request account deletion via `DELETE /users/me`, then the system soft-deletes the account, schedules image cleanup from storage within 30 days, and sends confirmation email; Stripe subscriptions are canceled via API
- Given the API receives requests, when a user has revoked LGPD consent but still has an active session, then image upload and render generation endpoints return HTTP 403 with message "Consentimento LGPD necessario para processar imagens"

## Technical Notes

### PRD Traceability
- **NFR-08:** LGPD compliance with explicit consent for image processing
- **NFR-09:** Images not used for training without explicit opt-in
- **NFR-17:** Disclaimer "imagem ilustrativa gerada por IA" on all generated images

### Architecture
- **Consent Storage:** Add `lgpd_consent_at` (timestamptz), `lgpd_consent_version` (text), `training_opt_in` (boolean default false) columns to `profiles` table
- **Middleware:** New `requireLgpdConsent` middleware on image processing routes (upload, render, edit)
- **UI Disclaimer:** Reusable `<AiDisclaimer />` component applied to all render display points
- **Data Export:** Async job that aggregates user data from profiles, projects, renders, subscriptions tables
- **Account Deletion:** Soft-delete with 30-day retention before permanent purge (Storage cleanup via cron/scheduled function)

### LGPD Reference
- Art. 7, I: Consentimento do titular
- Art. 8: Consentimento deve ser livre, informado e inequivoco
- Art. 18: Direitos do titular (acesso, correcao, exclusao, portabilidade)
- Art. 46: Medidas de seguranca
