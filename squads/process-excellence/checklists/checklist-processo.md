# Checklist de Qualidade de Processo

**Squad:** Process Excellence
**Uso:** Avaliar a qualidade e maturidade de um processo existente ou redesenhado.
**Instrucao:** Marque cada item como concluido [x] ou pendente [ ]. Itens nao aprovados
devem gerar acao corretiva antes de considerar o processo como maduro.

---

## 1. Mapeamento

- [ ] Fluxo do processo esta documentado (AS-IS ou TO-BE)
- [ ] Todos os atores/responsaveis estao identificados por etapa
- [ ] Tempos de processamento e espera estao registrados por etapa
- [ ] Handoffs entre atores/equipes estao claros e documentados
- [ ] Trigger de inicio (o que dispara o processo) esta definido
- [ ] Entregavel final (o que existe quando o processo termina) esta definido
- [ ] Pontos de decisao estao mapeados com criterios explicitos

## 2. Eficiencia

- [ ] Nao ha desperdicios evidentes (TIMWOODS aplicado)
- [ ] Gargalo do processo esta identificado (maior fila ou utilizacao)
- [ ] Atividades estao classificadas: Valor Agregado (VA) / Necessario sem Valor (RNVA) / Desperdicio (NVA)
- [ ] PCE (Process Cycle Efficiency) foi calculado: tempo VA / lead time total
- [ ] Etapas NVA tem plano de eliminacao ou justificativa documentada
- [ ] Etapas RNVA tem plano de minimizacao ou automacao

## 3. Conformidade

- [ ] SOP (Procedimento Operacional Padrao) existe para o processo
- [ ] SOP esta atualizada (revisada nos ultimos 6 meses)
- [ ] O processo e seguido na pratica conforme documentado
- [ ] Desvios conhecidos estao documentados com justificativa
- [ ] Existe responsavel definido (process owner) pelo processo
- [ ] Requisitos regulatorios/legais aplicaveis estao identificados e atendidos

## 4. Metricas

- [ ] KPIs do processo estao definidos (3 a 5 indicadores)
- [ ] Cada KPI tem formula de calculo e fonte de dados
- [ ] Baseline foi medido (minimo 30 dias de dados reais)
- [ ] Metas SMART estao estabelecidas para cada KPI
- [ ] Dashboard ou relatorio de acompanhamento esta disponivel
- [ ] Cadencia de revisao esta definida (diaria/semanal/mensal)
- [ ] Protocolo de acao para KPI fora da meta existe

## 5. Automacao

- [ ] Tarefas repetitivas e baseadas em regras estao identificadas
- [ ] Automacoes viaveis estao mapeadas com ferramenta sugerida
- [ ] ROI foi calculado para cada automacao proposta
- [ ] Automacoes existentes estao funcionando corretamente
- [ ] Nenhuma automacao foi aplicada sobre processo com desperdicio nao eliminado

## 6. Gestao de Mudanca

- [ ] Stakeholders afetados pela mudanca estao informados
- [ ] Treinamento foi realizado para os executores do processo
- [ ] Material de apoio (SOP, FAQ, troubleshooting) esta acessivel
- [ ] Adocao esta sendo medida (% de execucoes conforme novo processo)
- [ ] Canal de feedback para duvidas e problemas esta ativo
- [ ] Revisao pos-implementacao esta agendada (30-60 dias apos mudanca)
