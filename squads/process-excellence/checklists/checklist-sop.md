# Checklist de Qualidade de SOP

**Squad:** Process Excellence
**Uso:** Avaliar a qualidade de um Procedimento Operacional Padrao (SOP) antes de publicar.
**Instrucao:** Marque cada item como concluido [x] ou pendente [ ]. Uma SOP so deve ser
publicada quando todos os itens obrigatorios estiverem aprovados.

---

## 1. Estrutura

- [ ] Objetivo do procedimento esta claro em 1-2 frases
- [ ] Escopo esta definido (o que esta incluso e o que esta fora)
- [ ] Pre-requisitos estao listados (ferramentas, acessos, materiais, conhecimentos)
- [ ] Resultado esperado esta descrito (o que existe quando a SOP e concluida)
- [ ] Publico-alvo esta identificado (quem vai executar este procedimento)

## 2. Conteudo

- [ ] Passos estao numerados em sequencia logica
- [ ] Cada passo contem exatamente 1 acao (nunca 2 acoes no mesmo passo)
- [ ] Todos os passos comecam com verbo de acao (abrir, clicar, enviar, preencher)
- [ ] Pontos de decisao estao explicitados com criterios (se X, faca Y; se Z, faca W)
- [ ] Resultados esperados estao indicados apos cada etapa critica (o que o executor deve ver/obter)
- [ ] Nenhum passo assume conhecimento implicito do leitor

## 3. Acessibilidade

- [ ] Linguagem e simples e direta (nivel de leitura: ensino medio)
- [ ] Termos tecnicos e siglas estao explicados na primeira ocorrencia
- [ ] Um iniciante sem experiencia previa consegue executar seguindo apenas o texto
- [ ] Capturas de tela ou diagramas estao incluidos onde a descricao textual nao basta
- [ ] Formatacao facilita a leitura rapida (listas, negritos, espacamento)

## 4. Robustez

- [ ] Secao de troubleshooting existe com problemas comuns por etapa
- [ ] FAQ esta incluido com pelo menos 3 perguntas frequentes
- [ ] Cenarios de erro estao cobertos (o que fazer se algo der errado)
- [ ] Contato de suporte ou escalamento esta indicado para casos nao cobertos
- [ ] Limites e restricoes do procedimento estao documentados

## 5. Manutencao

- [ ] Numero de versao esta presente no documento
- [ ] Data da ultima revisao esta registrada
- [ ] Responsavel pela manutencao da SOP esta definido (nome ou cargo)
- [ ] Proxima data de revisao esta agendada
- [ ] Historico de alteracoes esta documentado (changelog)
